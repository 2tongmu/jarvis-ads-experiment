# ADS Schematic Connectivity Checker

Parses an ADS-generated netlist (from `sch.generate_netlist()`) and verifies all nodes are
properly connected. Catches wiring bugs before opening in ADS GUI.

## When to Use

Always run after `generate_netlist()` in Stage 3 of the PDK pipeline. A schematic that builds
without Python errors can still have broken connections — ADS places components silently even
if wire endpoints don't land on pin snap_points.

## Check Script

Use `scripts/check_netlist.py`. Run it on any `.net` file generated by ADS:

```bash
python scripts/check_netlist.py path/to/generated.net
```

Or inline after generate_netlist():

```python
import subprocess
result = subprocess.run(
    ["python", "check_netlist.py", str(ads_net_path)],
    capture_output=True, text=True
)
print(result.stdout)
```

## What It Checks

### 1. Floating nodes (appear in only 1 component)
A node connected to only one component pin = open circuit. This is the most common symptom
of a wire endpoint missing a pin snap_point.

### 2. Signal path continuity (Term1 → Term2)
Traces the connected graph from Port/Term1 through all series elements to Port/Term2.
Fails if any gap breaks the path.

### 3. FET pin coverage
Each PDK FET (PP1029_CPW_PDK, PP1029_MS_PDK) should have all 3 ports (gate, drain, source)
appearing in ≥2 nets. A pin appearing in only 1 net = floating gate, open drain, etc.

### 4. Shunt paths to GND
Any component with one pin on node `0` (GND) should have its other pin connected to the
signal path (not floating).

### 5. GND node (node `0`) connectivity
GND should appear in many components. If it appears in fewer than expected, some GROUND
instances are likely not connected.

## Pass/Fail Output

```
[CHECK] Parsing: spdt_ads_import_generated.net
[CHECK] Components: 24  |  Nodes: 31

[PASS] No floating nodes detected
[PASS] Signal path Term1 → Term2 connected (17 hops)
[PASS] All FET pins connected (Q1a, Q3a, Q1b, Q3b)
[PASS] All shunt paths reach GND
[PASS] GND node appears in 12 components

ALL CHECKS PASSED ✅
```

On failure:
```
[FAIL] Floating node N__1 -- only appears in: Term1
[FAIL] Signal path broken after: R:Rpad_in (N__0 not reachable from N__1)
[FAIL] Q3a Source pin floating (node N__14 appears in 1 component only)

3 CHECKS FAILED ❌
```

## Common Failure → Root Cause Map

| Failure | Likely cause |
|---|---|
| Term1 floating | Wire from Term1 to Rpad_in missing; Term placement at wrong x |
| Rtrm→Lrt gap | wire() call missing between shunt R P2 and L P1 |
| FET drain/source floating | CPW instance at wrong (cx,cy); drain_x offset wrong |
| Co1a on wrong node | Co1a placed before Q1b source instead of after |
| Many floats | angle= wrong; P1/P2 offsets inverted |

## ADS Node Naming Note

`generate_netlist()` assigns opaque names like `N__1`, `N__3` etc. — not the logical names
from the hand-written netlist. This is normal. The checker works on structural connectivity,
not node name matching.

## Integration with ads-netlist-translator

After Stage 3 (`ads_import_netlist.py`) calls `generate_netlist()` and saves the .net file,
immediately run the connectivity check. If any check fails:
1. Print the failure details
2. Do NOT report success to the parent session
3. Fix the placement/wiring bug in the build script
4. Rebuild and re-check until all pass
